
# Credit Card Fraud Detection — Random Forest

Проект по обнаружению мошеннических транзакций с кредитными картами с использованием алгоритма Random Forest. Особенность задачи — сильный дисбаланс классов: только **0.17%** транзакций являются мошенническими.

---

## Датасет

[Kaggle — Credit Card Fraud Detection](https://www.kaggle.com/datasets/mlg-ulb/creditcardfraud)

- **284 807** транзакций по кредитным картам (Европа, сентябрь 2013)
- **492** мошеннических транзакции (0.17%)
- Признаки V1–V28 анонимизированы через PCA
- Признаки Time (время) и Amount (сумма) — в исходном виде
- Целевая переменная: Class (0 = нормальная, 1 = мошенничество)

---

## Задача

Сравнить три стратегии борьбы с дисбалансом классов и выбрать лучшую:

| Стратегия | Описание |
|---|---|
| `class_weight='balanced'` | Штрафуем модель сильнее за пропуск мошенника |
| Undersampling | Уменьшаем количество нормальных транзакций до уровня мошеннических |
| SMOTE | Генерируем синтетические мошеннические транзакции |

---

## Результаты

| Стратегия | ROC-AUC | Precision | Recall | Avg Precision |
|---|---|---|---|---|
| class_weight | 0.9747 | **84.2%** | 81.6% | **82.9%** |
| undersampling | 0.9788 | 4.2% | **91.8%** | 69.8% |
| **SMOTE** | **0.9846** | 61.0% | 87.8% | 82.1% |

**Лучшая стратегия по ROC-AUC — SMOTE (0.9846)**

Ключевой признак во всех трёх моделях — **V14** (важность ~0.18–0.21). Сумма транзакции (Amount) оказалась малозначимой — мошенники не выделяются размером платежа.

## Метрики оценки

- **ROC-AUC** — общее качество разделения классов
- **Precision** — точность: сколько из заблокированных реально мошенники
- **Recall** — полнота: сколько мошенников из всех реальных поймали
- **Avg Precision** — площадь под Precision-Recall кривой
- **Confusion Matrix** — матрица ошибок
- **Calibration Curve** — калибровка вероятностей модели

---

## Выводы

- **SMOTE** — лучший выбор по общему качеству (ROC-AUC 0.985, Recall 87.8%)
- **class_weight** — лучший выбор если важно минимизировать ложные блокировки (Precision 84.2%)
- **Undersampling** — не рекомендуется для продакшена: 2 040 ложных блокировок на 56 962 транзакции

В реальном банке пропущенный мошенник обходится дороже чем временная блокировка карты — поэтому предпочтительнее **SMOTE** с высоким Recall.
